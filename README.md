# Nexts Hub

The unified content repository for Nexts: **skills / plugins / assistants / MCP servers** — four kinds of distributable content, one repo, one directory per category.

中文文档: [README.zh-CN.md](README.zh-CN.md)

```
nexts-hub/
├── index.json      Top-level manifest: per-category versions and entry points; the app uses it to check for updates
├── package.json    Unified tooling entry (npm scripts: scaffold / sync / validate)
├── skills/         Skills (imported from nexts-cc/skills, internal layout unchanged)
│   └── skills/<name>/SKILL.md
├── plugins/        Plugins (imported from nexts-cc/plugins, internal layout unchanged)
│   ├── .agents/plugins/marketplace.json   Authoritative plugin registry
│   └── plugins/<name>/.nexts-plugin/plugin.json
├── assistants/     Assistants (one directory = one assistant)
│   └── <id>/assistant.json
└── mcp/            MCP server registrations (one directory = one server)
    └── <id>/mcp.json
```

Each category is registered in a `.agents` marketplace manifest (`skills/.agents/skills/marketplace.json`, `plugins/.agents/plugins/marketplace.json`, `assistants/.agents/assistants/marketplace.json`, `mcp/.agents/mcp/marketplace.json`). The plugins one is authoritative and hand-maintained; the other three are generated by `npm run sync`.

## Distribution model

Content flows from this repo into the app data directory (`%APPDATA%\NextsAI\`):

| Category | Installs to |
|----------|-------------|
| skills | `%APPDATA%\NextsAI\skills\<name>\` |
| plugins | `%APPDATA%\NextsAI\plugins\<name>\` |
| assistants | `%APPDATA%\NextsAI\assistants\<id>\` |
| mcp | `%APPDATA%\NextsAI\mcp\<id>\` (or merged into the nextcli config) |

Two delivery paths, either or both:

1. **Bundled at package time** — the installer ships this repo (or selected categories) inside the app resources and seeds the data directory on first launch.
2. **Online updates** — the app pulls this repo from GitHub, compares versions in `index.json`, and updates the data directory incrementally.

Content in the data directory is a **runtime copy**. User-created content (custom assistants, agent-written site patterns, …) lives alongside pulled content and must never be overwritten by updates.

## Maintenance

- New skill: `skills/skills/<name>/SKILL.md` (see `skills/templates/`).
- New plugin: `plugins/plugins/<name>/`, registered in `plugins/.agents/plugins/marketplace.json`.
- New assistant: `assistants/<id>/assistant.json` (schema in `assistants/README.md`).
- New MCP server: `mcp/<id>/mcp.json` (schema in `mcp/README.md`).
- Prefer the scaffolding scripts: `npm run new:skill -- <id>` (likewise `new:plugin` / `new:assistant` / `new:mcp`), then `npm run sync` and `npm run validate`. See `docs/DEVELOPMENT.md`.
- On every content release, **bump the matching category `version` in `index.json`** — the app relies on it to decide whether to update.
- Encoding: UTF-8 without BOM (especially for files containing Chinese).
